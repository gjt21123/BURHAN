# Video recording checklist

- [ ] Start `npm run dev` before recording; do not run a live provider command.
- [ ] Use a 16:9 browser window at 100% zoom with readable labels.
- [ ] Follow [video-shot-list.md](video-shot-list.md) and [video-voiceover.md](video-voiceover.md).
- [ ] Keep total duration below three minutes.
- [ ] Show `LIVE CODEX RUN`, `LIVE BURHAN VERIFICATION`, and `DETERMINISTIC REPAIR DEMO` exactly as labeled.
- [ ] Explain `REPAIR_CONTEXT_UNAVAILABLE` only as the retained-context limitation of the original live run.
- [ ] Never claim live GPT-5.6 inference or a live Codex repair.
- [ ] Do not display credentials, environment values, prompts, private reasoning, hidden validator source, protected paths, raw JSONL, or raw stderr.
- [ ] Verify reset returns the display to the original `REJECTED` state.
- [ ] Review final audio, captions, and every visible label before upload.
