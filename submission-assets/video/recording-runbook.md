# Recording runbook

1. Run `npm run dev` or start the built app locally. Open the app in a clean 16:9 browser window at 100% zoom.
2. Begin with the hero state. Read the first paragraph from `final-voiceover.txt` and click `Compile Proof Contract`.
3. Review and seal the contract while reading the second paragraph.
4. Hold the `LIVE CODEX RUN` card while reading the third paragraph; then hold `LIVE BURHAN VERIFICATION / REJECTED` for the fourth.
5. Show the sanitized counterexample, read the fifth paragraph, and click `APPROVE REPAIR` once.
6. Show `DETERMINISTIC REPAIR DEMO`, read the sixth paragraph, then hold `FRESH VERIFICATION / VERIFIED` and linked receipts while reading the seventh.
7. Read the final paragraph, click `RESET DEMO`, and wait for `RESET COMPLETE`.

If any action, narration, or label is missed: stop the take, reset to the original rejected state, and re-record the entire take. Do not splice in fake provider activity. Record narration only when a real audio track is being captured.
